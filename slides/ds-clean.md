# What is a Database?

Formally, a   __database__   refers to a set of related data and the way it is organized.  _Access_  to this data is usually provided by a " _database_   _management_   _system_ " ( _DBMS_ ) consisting of an integrated set of computer  _software_  that allows users to  _interact_  with one or more databases and provides  _access_  to all of the data contained in the database.

The DBMS provides  _various_   _functions_  that allow entry, storage and retrieval of large quantities of information and provides ways to  _manage_  how that information is organized.

Because of the  _close_   _relationship_  between them, the term   __database__   is often used casually to  _refer_  to  _both_  a  _database_  and the  _DBMS_  used to manipulate it.

Key/Value Stores are the simplest NOSQL database model. They can just store pairs of keys and values and retrieve a value for a given key. Due to the simplicity of the model, they can ensure very high performance and scalability.

Document stores are designed to handle semi-structured data like XML or JSON. The main difference with the key/value stores is that the documents are processed and their internal structures can be used for indexing and other optimization purposes.

In a Graph database, data is represented as a graph in which each node is a record and each arc is a relationship between two nodes. They are optimized to represent complex data models

Wide Column Stores store data in tables, rows, and dynamic columns. They provide a lot of flexibility over relational databases because each row is not required to have the same columns. They are used by popular social networks like Twitter and Facebook to store content produced by their users.

---

Graph — Graph databases are used to store and query highly connected data. Data can be modeled in the form of entities (also referred to as nodes, or vertices) and the relationships between those entities (also referred to as edges). The strength or nature of the relationships also carry significant meaning in graph databases.

Users can then traverse the graph structure by starting at a defined set of nodes or edges and travel across the graph, along defined relationship types or strengths, until they reach some defined condition. Results can be returned in the form of literals, lists, maps, or graph traversal paths. Graph databases provide a set of query languages that contain syntax designed for traversing a graph structure, or matching a certain structural pattern.

The network databases like CODSASYL are still more or less based on a hierarchical data model, thinking in terms of parent-child (or owner-member in CODASYL terminology) relationships. This also means that in network database you can't relate arbitrary records to each other. For example, you may use a graph database to analyze what relationships exist between entities.

Wide-column — A wide column data store is a type of NoSQL database that stores data in columns rather than rows, making it highly scalable and flexible. In a wide column data store, data is organized into column families, which are groups of columns that share the same attributes. Each row in a wide column data store is identified by a unique row key, and the columns in that row are further divided into column names and values.

Unlike traditional relational databases, which have a fixed number of columns and data types, wide column data stores allow for a variable number of columns and support multiple data types. The most significant benefit of having column-oriented databases is that you can store large amounts of data within a single column. This feature allows you to reduce disk resources and the time it takes to retrieve information from it.

# What is Data Analytics?

Data analytics draws from a range of disciplines — including   __computer__   __ __   __programming__,   __mathematics__, and   __statistics__   — to perform analysis on data in an effort to  _describe_,  _predict_, and  _improve_  performance.

To ensure robust analysis,   __data analytics teams leverage a range of data management techniques__, including data  _mining_, data  _cleansing_, data  _transformation_  and data  _modeling_.

Definition:
‘Data analytics is a discipline focused on extracting insights from data. It comprises the processes, tools and techniques of data analysis and management, including the collection, organization, and storage of data.' 

4 Types of data analytics
Descriptive analytics: What has happened and what is happening right now? Descriptive analytics uses historical and current data from multiple sources to describe the present state by identifying trends and patterns. In business analytics, this is the purview of business intelligence (BI).

Diagnostic analytics: Why is it happening? Diagnostic analytics uses data (often generated via descriptive analytics) to discover the factors or reasons for past performance.

Predictive analytics: What is likely to happen in the future? Predictive analytics applies techniques such as statistical modeling, forecasting, and machine learning to the output of descriptive and diagnostic analytics to make predictions about future outcomes. Predictive analytics is often considered a type of "advanced analytics," and frequently depends on machine learning and/or deep learning.

Prescriptive analytics: What do we need to do? Prescriptive analytics is a type of advanced analytics that involves the application of testing and other techniques to recommend specific solutions that will deliver desired outcomes. In business, predictive analytics uses machine learning, business rules, and algorithms.

# Basic Concepts in Probability (1/5)

 __Random experiment __  is an action for which  _all possible outcomes_  can be listed, but for which the  _actual_  outcome on any given trial of the experiment  _cannot_  be  _predicted_  with  _certainty_.

* For example, a coin has two sides and when tossed will land with either the "head" side (H) up or the "tail" side (T) up, but not both. Or, a die has six sides and when thrown will land with either the 1, 2, 3, 4, 5, or 6 side up.

 __Sample space __  associated with a random experiment is  _the set of all possible outcomes _ (H or T with the  _coin_; or 1, 2, 3, 4, 5, or 6 with the  _die_ ).

 __Probability of an__       __outcome __  is a number p between 0 and 1 that measures the likelihood that this outcome will occur on a single trial. The value  _p = 0_  corresponds to the outcome e being  _impossible_  and the value  _p = 1_  corresponds to the outcome e being  _certain_.

 __Set __  is a defined  _collection of items_. For example, the number rolled is odd {1,3,5} or even {2,4,6}.

 __Event__, a set of outcomes is an   __event__   __. __ The   __probability of an event __  is the sum of the probabilities of the individual outcomes of which it is composed. It is denoted P(A).

_For example_, the events   __"odd-numbered rolls" __  and   __"even-numbered rolls" __  are mutually exclusive and there are  _no elements common to both of them_.

In, decision theory and artificial intelligence, a rational agent is an agent that has clear preferences, models uncertainty via expected values of variables or functions of variables, and always chooses to perform the action with the optimal expected outcome for itself from among all feasible actions.

1. Agente riflesso semplice
Gli agenti riflessi semplici prendono decisioni basandosi esclusivamente sull'input corrente, senza considerare i risultati passati o potenziali futuri. Reagiscono direttamente alla situazione corrente senza stato interno o memoria.

Esempio: un termostato che accende il riscaldamento quando la temperatura scende sotto una certa soglia, ma non tiene conto delle precedenti letture della temperatura o delle previsioni meteo a lungo termine - fig Bonaire Gas Heater Overheat Switch 120 Deg. 5431606SP

Caratteristiche dell'agente riflesso semplice:
Reattivo: reagisce direttamente agli input sensoriali attuali senza considerare le esperienze passate o le conseguenze future.
Ambito limitato: in grado di gestire compiti o ambienti semplici con relazioni causa-effetto chiare.
Risposta rapida: prende decisioni rapide basandosi esclusivamente sulla situazione attuale, consentendo una rapida esecuzione delle azioni.
Mancanza di adattabilità: incapacità di apprendere o adattarsi in base al feedback, il che lo rende meno adatto ad ambienti dinamici o in continua evoluzione.

2. Agenti riflessi basati su modelli
Gli agenti riflessi basati su modelli migliorano gli agenti riflessi semplici incorporando rappresentazioni interne dell'ambiente. Questi modelli consentono agli agenti di prevedere i risultati delle loro azioni e prendere decisioni più informate. Mantenendo stati interni che riflettono aspetti inosservati dell'ambiente e utilizzando percezioni passate, questi agenti sviluppano una comprensione completa del mondo. Questo approccio li equipaggia per navigare efficacemente in ambienti complessi, adattarsi a condizioni mutevoli e gestire l'osservabilità parziale.

Esempio: un sistema di guida autonoma non solo reagisce alle condizioni stradali attuali, ma tiene conto anche della conoscenza delle regole del traffico, delle mappe stradali e delle esperienze passate per navigare in sicurezza.

Caratteristiche Agenti riflessi basati sul modello
Adattivo: mantiene un modello interno dell'ambiente per anticipare stati futuri e prendere decisioni informate.
Comprensione contestuale: considera sia gli input correnti sia i dati storici per determinare le azioni appropriate, consentendo un processo decisionale più articolato.
Sovraccarico computazionale : richiede risorse per creare, aggiornare e utilizzare il modello interno, con conseguente aumento della complessità computazionale.
Prestazioni migliorate: può gestire attività e ambienti più complessi rispetto ai semplici agenti riflessi, grazie alla sua capacità di incorporare esperienze passate.

3. Agenti basati sugli obiettivi
Gli agenti basati sugli obiettivi hanno obiettivi predefiniti o traguardi che mirano a raggiungere. Combinando descrizioni di obiettivi e modelli dell'ambiente, questi agenti pianificano di raggiungere obiettivi diversi, come raggiungere destinazioni particolari. Utilizzano metodi di ricerca e pianificazione per creare sequenze di azioni che migliorano il processo decisionale al fine di raggiungere gli obiettivi. Gli agenti basati sugli obiettivi differiscono dagli agenti riflessi perché includono processi decisionali lungimiranti e orientati al futuro.

Esempio : un robot per le consegne incaricato di consegnare pacchi in luoghi specifici. Analizza la sua posizione attuale, la destinazione, i percorsi disponibili e gli ostacoli per pianificare un percorso ottimale verso la consegna del pacco.

Caratteristiche degli agenti basati sugli obiettivi:
Intenzionale: opera con obiettivi predefiniti, fornendo una direzione chiara per il processo decisionale e la selezione delle azioni.
Pianificazione strategica: valuta le azioni disponibili in base al loro contributo al raggiungimento degli obiettivi, ottimizzando il processo decisionale per il loro raggiungimento.
Definizione delle priorità degli obiettivi: è possibile stabilire le priorità degli obiettivi in ​​base alla loro importanza o urgenza, consentendo un'allocazione efficiente delle risorse e degli sforzi.
Flessibilità degli obiettivi: capacità di adattare gli obiettivi o di modificare le strategie in risposta ai cambiamenti dell'ambiente o alle nuove informazioni.

4. Agenti basati sull'utilità
Gli agenti basati sull'utilità vanno oltre i metodi di base orientati agli obiettivi tenendo conto non solo del raggiungimento degli obiettivi, ma anche della qualità dei risultati. Utilizzano funzioni di utilità per valutare vari stati, consentendo confronti dettagliati e compromessi tra diversi obiettivi. Questi agenti ottimizzano la soddisfazione complessiva massimizzando l'utilità attesa, considerando le incertezze e l'osservabilità parziale in ambienti complessi. Anche se il concetto di agenti basati sull'utilità può sembrare semplice, la loro implementazione efficace comporta una modellazione complessa dell'ambiente, della percezione, del ragionamento e dell'apprendimento, insieme ad algoritmi intelligenti per decidere il miglior corso d'azione di fronte alle sfide computazionali.

Esempio: un algoritmo di un consulente finanziario suggerisce opzioni di investimento prendendo in considerazione fattori quali potenziali rendimenti, tolleranza al rischio e requisiti di liquidità, con l'obiettivo di massimizzare la soddisfazione finanziaria a lungo termine dell'investitore.
